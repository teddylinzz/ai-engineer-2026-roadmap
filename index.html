Frontier Lab Interview Question

Q: Explain GQA and how it impacts KV cache memory sizing in large scale LLM serving.

In standard Multi-Head Attention (MHA), KV heads equal Query heads ($H_{kv} = H_q$), causing KV cache to scale linearly and choke GPU memory at long context lengths. Grouped-Query Attention (GQA) groups query heads into key-value head clusters (e.g. 8:1 ratio in Llama 3 8B), reducing the stored KV cache size by 8x while retaining near-identical model output accuracy. This allows larger batch sizes and much longer context deployment on standard hardware.
